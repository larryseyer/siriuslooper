# S2 — OttoPane tab via `OTTOProcessor::createEditor()` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land a top-level "OTTO" tab in IDA that hosts OTTO's full editor (returned by `OTTOProcessor::createEditor()`). After this slice the operator clicks the OTTO tab, sees OTTO's full UI, hits Play, and hears audio through master — closing the M-OTTO-4 audibility gap.

**Architecture:** S1 (IDA `c10cfc8` / OTTO `b7654144`) already embeds `std::unique_ptr<OTTOProcessor>` inside `OttoHost::Impl`. S2 adds: (a) a `juce::AudioProcessor& OttoHost::getProcessor()` accessor; (b) a new `ida::OttoPane` `juce::Component` in `app/` that owns a `std::unique_ptr<juce::AudioProcessorEditor>` returned by `processor.createEditor()`; (c) an `OttoHost` → `OTTOProcessor::setEmbeddedInHost(true)` call at construction so the editor's existing `isPluginMode_` gate hides its unified TransportBar (IDA's own TransportBar lands in S3); (d) skipping the `OutputRoutingSection` in `PreferencesDialog` when embedded. The OTTO-side change is a cross-project edit per the inbox protocol — landed first, then the IDA atomic commit references it.

**Tech Stack:** JUCE `AudioProcessorEditor`, `juce::Component`, `juce::TabbedComponent`. C++20. Catch2 for headless tests. Operator-verified GUI per `CLAUDE.md` ("GUI changes are operator-verified, not unit-tested").

```cpp
#include "OttoPane.h"

#include "ida/OttoHost.h"

#include <juce_audio_processors/juce_audio_processors.h>

namespace ida
{

OttoPane::OttoPane (OttoHost& host)
{
    // OttoHost::Impl::Impl() has already called
    // OTTOProcessor::setEmbeddedInHost(true), so the editor's isPluginMode_
    // derivation picks up the embedded path during construction below.
    editor_.reset (host.getProcessor().createEditor());

    if (editor_ != nullptr)
        addAndMakeVisible (*editor_);
}

OttoPane::~OttoPane() = default;

void OttoPane::resized()
{
    if (editor_ == nullptr)
        return;

    // OTTOEditor's desktop default is 1200×1200, min 600×600, resizable.
    // The pane stretches the editor to fill the tab area — OTTOEditor's
    // own ResizableCornerComponent / size-limits constraints in the
    // standalone window do not apply when it is a child Component, so a
    // straight setBounds is safe (and matches how AUv3 hosts embed it).
    editor_->setBounds (getLocalBounds());
}

void OttoPane::paint (juce::Graphics& g)
{
    // The editor paints the entire surface when present. The fallback
    // background appears only if createEditor returned nullptr — a state
    // §4.2 of the integration spec documents as "OTTO UI failed to
    // initialize" — surfacing the empty tab is the operator's signal to
    // file a bug.
    if (editor_ == nullptr)
    {
        g.fillAll (juce::Colours::black);
        g.setColour (juce::Colours::red);
        g.setFont (juce::Font (juce::FontOptions (16.0f)));
        g.drawText ("OTTO UI failed to initialize",
                    getLocalBounds(), juce::Justification::centred);
    }
}

} // namespace ida
```
