# LMC Calibration Corruption Recovery (M8 S6) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** On session load (and at startup), validate a device-scoped calibration sidecar's SHA-256 checksum; on corruption/absence, reset to identity calibration, post a Warning notification, and persist a `recalibrationPending` marker the future loopback engine will consume.

**Architecture:** A pure, JUCE-free `CalibrationStore` module (string in / string out, exhaustively unit-testable) handles serialize + parse-and-validate of a calibration document (`AudioDeviceCalibration` + `recalibrationPending`) carrying a SHA-256 over a canonical text payload. A caller-side `postCalibrationRecoveryNotification` helper announces a non-`Ok` result, parallel to `postConstituentStateNotifications` (M8 S3). `MainComponent` owns the device-scoped sidecar file I/O and wires startup + on-load validation. `AudioDeviceCalibration` is NOT modified.

**Tech Stack:** C++20, CMake/Ninja, Catch2 (`IdaTests`). Engine public API stays JUCE-free (`engine/CMakeLists.txt`).

**Spec:** `docs/superpowers/specs/2026-05-20-m8-s6-design.md`

## Task 1: `CalibrationStore` serialize + parse/validate (pure)

**Files:** create `CalibrationStore.h`, `CalibrationStore.cpp`, `tests/CalibrationRecoveryTests.cpp`; modify `engine/CMakeLists.txt`, `tests/CMakeLists.txt`.

### Canonical document format

```
rateFactor=<num>/<den>
offset=<num>/<den>
pending=<0|1>
sha256=<64-hex>
```

Payload = the first three lines including the trailing `\n` after `pending=...`. `sha256=` is computed over the payload bytes; the checksum line is not part of the hashed region.

### Header API (`engine/include/ida/CalibrationStore.h`)

```cpp
#pragma once

#include "ida/AudioDeviceCalibration.h"
#include "ida/INotificationSink.h"

#include <string>

namespace ida
{

enum class CalibrationLoadStatus { Ok, Missing, Corrupt };

struct CalibrationDocument
{
    AudioDeviceCalibration calibration { AudioDeviceCalibration::identity() };
    bool recalibrationPending { false };
};

/// Canonical payload + SHA-256 wrapper. Pure.
std::string serializeCalibration (const CalibrationDocument& doc);

struct CalibrationParseResult
{
    CalibrationLoadStatus status;
    CalibrationDocument   document; // {identity, pending=true} unless status==Ok
};

/// Empty input -> Missing. Malformed / out-of-range / checksum-mismatch -> Corrupt.
/// On any non-Ok status the returned document is {identity, pending=true}. Pure.
CalibrationParseResult parseAndValidateCalibration (const std::string& fileContents);

/// No-op on Ok. On Missing/Corrupt posts one Warning/StateRepair naming the
/// cause + recovery. Kept out of the parse so the parse stays sink-free.
void postCalibrationRecoveryNotification (CalibrationLoadStatus status,
                                          INotificationSink&    sink);

} // namespace ida
```

`postCalibrationRecoveryNotification` lands here but is pinned by Task 2's test (implemented now, tested next — same as M8 S4 Task 2/3 where the impl shipped in Task 1).

### Implementation notes (`engine/src/CalibrationStore.cpp`)

- Serialize a `Rational` as `num + "/" + den` via `numerator()`/`denominator()` (already normalized).
- `sha256Hex` takes `std::span<const std::byte>`; build it from the payload `std::string` via `reinterpret_cast<const std::byte*>(payload.data()), payload.size()`.
- Parse: split into lines; require the four keys in order with the exact prefixes. A `recoverable()` helper returns `{ status, {AudioDeviceCalibration::identity(), true} }` for the non-Ok exits.
- Rational parse: split the value on `/`, parse two `int64` via `std::from_chars` / `std::stoll` (catch exceptions → Corrupt); zero denominator → Corrupt; reconstructing `AudioDeviceCalibration` with a non-positive rate factor throws `std::invalid_argument` → catch → Corrupt.
- Empty / whitespace-only `fileContents` → `Missing`.
- Recompute the payload hash, compare to the parsed `sha256=` value (case-insensitive not required — `sha256Hex` emits lowercase; serialize emits what it computes) → mismatch → Corrupt.

### Tests (`tests/CalibrationRecoveryTests.cpp`, tag `[calibration-recovery]`)

```cpp
#include "ida/CalibrationStore.h"
#include "ida/INotificationSink.h"

#include <catch2/catch_test_macros.hpp>

using namespace ida;
```

1. **Round-trip non-identity.** `CalibrationDocument` with `AudioDeviceCalibration(Rational(999987,1000000), Rational(1,48000))`, `pending=false` → `serializeCalibration` → `parseAndValidateCalibration` → `status==Ok`; `rateFactor()`, `offsetSeconds()` equal originals exactly; `pending==false`.
2. **Pending round-trips.** `{identity, pending=true}` → `Ok`, `pending==true`.
3. **Tampered payload → Corrupt.** Serialize, mutate a digit in the `rateFactor=` line, parse → `Corrupt`; document is `{identity (rateFactor 1/1, offset 0/1), pending=true}`.
4. **Tampered checksum → Corrupt.** Serialize, flip a hex char in `sha256=`, parse → `Corrupt`.
5. **Malformed → Corrupt.** Inputs: missing the `offset=` line; `rateFactor=abc/1`; `rateFactor=1/0`; `rateFactor=0/1` (non-positive) — each `Corrupt`.
6. **Empty → Missing.** `""` and `"   \n"` → `Missing`, document `{identity, pending=true}`.
