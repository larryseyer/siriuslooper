# M8 S3 — Constituent state machine + render-as-silence

## Problem

The Constituent graph can come to contradict itself. A loop may reference a
tape segment that no longer exists; a child's anchor or bounds may place it
outside its parent. Whitepaper §17.7 requires that such a Constituent is
**marked, not deleted** — it renders as silence, the performer is informed,
and its identity, metadata, and content survive so the performer can repair
rather than recreate. Today `Constituent` is purely structural: there is no
notion of validity, and the render path produces audio for any node whose
placement span is active, regardless of whether its references hold.

This session adds the failure half of M8: a `Valid | Broken | Invalid` state
derived from the graph, and a render path that treats Broken and Invalid
nodes (and everything nested inside them) as silence.

## Scope

In:

- `Broken` — a leaf loop whose tape reference does not resolve to content.
- `Invalid` — a Constituent whose conceptual span is not contained within its
  parent's local time frame (anchor/bounds contradict parent, V7 §17 line
  567 / whitepaper §17.7 line 1713).
- Render-as-silence for both states, including the entire subtree of an
  offending node.
- A `Warning` notification on the existing `NotificationBus` when a node is
  found Broken or Invalid.

Out:

- **Tempo-map contradiction** (whitepaper §17.7 line 1711). Its semantics are
  revert-to-last-valid-map and mark-for-review, not render-as-silence; V7
  line 567 scopes Invalid to anchor/bounds only. Separate concern.
- **Real tape-reachability detection.** The TapeId→content manifest and the
  reachability scan are M8 S7–S8. S3 detects Broken through an injected
  resolver predicate; the real `TapeStore`-backed resolver wires in later.
- **Calibration recovery, tape rotation, WetCapture** — later M8 sessions.

## Key constraints

- `Constituent` is immutable; edits are copy-on-write and children are
  `std::shared_ptr<const Constituent>`. State therefore cannot be a mutable
  field on the node. It is **derived by a validation pass and never
  persisted** — the session JSON stores only structural intent, and state is
  recomputed at load and on edits.
- `RenderPipeline::activeReadsAt` is a hot path (queried per LMC time).
  Validation must be precomputed once and consulted by lookup, never run
  inside the render walk.

## Design

### `ConstituentState` — `core/include/ida/ConstituentState.h`

`enum class ConstituentState { Valid, Broken, Invalid };` — header-only, no
persistence, no dependencies.

### Tape-resolution seam

`using TapeResolver = std::function<bool(const TapeReference&)>;` — answers
"does this segment resolve to real content?" Injected into the validator so
the validator has no data-layer dependency. S3 ships the type plus a trivial
always-true default for callers without a store; the real resolver is S7–S8.

### `ConstituentValidator` — `engine/`

Placed in `engine/` (it needs the `TapeResolver` seam and feeds
`RenderPipeline`; `core/` stays dependency-light).

API:

```
ConstituentValidation validate(const Constituent& root,
                               const TapeResolver& resolver);
```

`ConstituentValidation` wraps `std::unordered_map<ConstituentId,
ConstituentState>` and exposes `state(ConstituentId) -> ConstituentState`
(default `Valid` for unknown ids) and `bool renderable(ConstituentId)`
(`== Valid`). It is immutable once built. A `std::hash<ConstituentId>`
specialization lives with the validation type if the id does not already
provide one.

The walk is depth-first and mirrors `RenderPipeline::collect`. Per node:

- **Broken** — node is a leaf loop with a `tapeReference` and
  `!resolver(*tapeReference)`.
- **Invalid** — span not contained in the parent's local frame:
  `child.conceptualIn() < 0` or
  `child.conceptualOut() > (parent.conceptualOut() − parent.conceptualIn())`.
  The root has no parent and is never Invalid by containment. The coordinate
  basis is confirmed against `engine/src/RenderPipeline.cpp` lines 40, 80, 86:
  a child's `conceptualIn/Out` are 0-based positions in the parent's local
  time frame.
- **Valid** — otherwise.
- A node satisfying both conditions reports **Invalid** (structural error
  dominates; recorded in a code comment).

The validator is pure: it returns the map and does not touch the bus.

### Render-as-silence — `engine/RenderPipeline`

`RenderPipeline` takes a `ConstituentValidation` (computed once at load / on
tree edit). In `collect`, before emitting any `ActiveRead` or descending:
if `validation.state(constituent.id()) != Valid`, return immediately — the
node emits nothing and its whole subtree is skipped, so the node's slot in
its parent is silence (§17.7). A default all-`Valid` `ConstituentValidation`
preserves current behavior for callers that have not adopted validation.

### Notification

The load path (caller) walks the validation result and posts a
`Warning`-level `NotificationBus` event per Broken/Invalid node, reusing the
M8 S1 drift-message formatting pattern in `engine/src/SessionSnapshot.cpp`,
with messages such as `constituent <id> broken: tape unresolved` and
`constituent <id> invalid: bounds outside parent`. Keeping notification in
the caller leaves the validator unit-testable in isolation.

## Tests — `tests/ConstituentStateTests.cpp`, tag `[constituent-state]`

1. Valid baseline — well-formed tree, always-true resolver → all `Valid`.
2. Broken — leaf loop + resolver returning false for its tape → that node
   `Broken`, siblings `Valid`.
3. Invalid (overflow end) — child `conceptualOut` past parent length.
4. Invalid (negative start) — child `conceptualIn` below 0.
5. Render-as-silence — feed validation into `RenderPipeline`; a Broken or
   Invalid node and a child nested under it produce no `ActiveRead` inside
   their span, while a Valid sibling still reads.
6. Identity preserved — the tree is structurally unchanged after validation
   (same ids, same shared children), pinned as in `ConstituentTests.cpp`.
7. Both-conditions precedence — broken-leaf that is also out-of-bounds
   reports `Invalid`.

Fixtures follow the inline `makeLeaf()` style in `ConstituentTests.cpp`.
