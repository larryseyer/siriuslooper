# M8 Session 1 — ArchivalMode enum + VersionPinning hashing

## Context

White paper §15.6 commits the architecture to a three-strategy contract
for handling non-deterministic plug-ins: *Determinism Contract*
(refuse / bypass plug-ins that fail to render bit-exact for identical
input + state), *Wet Capture* (store the rendered output as a parallel
parameter tape), and *Version Pinning* (record format-UID + version +
state-hash and warn the performer on reopen-time drift). White paper
line 1423 sets the default disposition: *"symbolic fidelity is never
optional. For non-deterministic plug-ins, the user chooses among the
three strategies above per session, per channel, or per plug-in
instance."* The V7 plan's M8 acceptance criterion narrows the default
to `VersionPinning` (line 563): the strategy that preserves symbolic
content unconditionally, costs no additional storage, and degrades
gracefully when plug-in binaries drift across machines or sessions.

The M7 out-of-process plug-in framework is shipped — the engine can
spawn `ida_plugin_host` children, route audio through them via
shared-memory rings, supervise failures, and host editors in
child-owned NSWindows. M8 builds the persistence + reporting surface
on top of that framework. This session is the smallest defensible
slice of M8: the `ArchivalMode` enum, the `VersionPinningRecord`
value type, the per-instance opt-in field, save/load round-trip
through the existing JSON session format, and the
`PluginVersionDrift` notification that surfaces reopen mismatches.

No CLAP-specific work lands here. No `Constituent` state-machine
work. No `WetCapture` writer. No tape reachability scan. Each of
those is its own later M8 session (per V7 plan lines 597-603). The
discipline of M8 is to land each strategy and each failure-semantics
piece in isolation, so the tests for each piece can be written
against a single new code path rather than against a moving target.

## Scope

In:

- `ArchivalMode` enum (three values, default `VersionPinning`).
- `VersionPinningRecord` value type carrying the persistable identity
  of one hosted plug-in instance.
- `PluginDescriptor::version` field for round-trip preservation of
  the format-reported version string.
- `EffectChainEntry::archivalMode` field defaulting to
  `VersionPinning`.
- SHA-256 helper reachable from `core/`, reusing the algorithm
  already linked into `persistence/` for `TapeStore`.
- Session save / load handling for `VersionPinningRecord` via the
  existing JSON session format.
- `NotificationBus::PluginVersionDrift` notification kind, emitted at
  load time when the recomputed record diverges from the persisted
  one.
- Tests covering the enum, the record round-trip, the no-drift
  reopen, and the drift-on-reopen path.

Out (deferred to later M8 sessions per V7 plan lines 597-603):

- CLAP `clap_plugin_state` integration for real state-blob hashing.
- `WetCapture` mode wiring (the tape writer + the
  `<channelId>.wet.tape` storage path).
- `DeterminismContract` enforcement — the render-twice verification
  pass that demotes a plug-in to non-deterministic if its outputs
  diverge for identical input + state.
- `Constituent::State` enum (`Valid | Broken | Invalid`) and the
  render-as-silence semantics.
- LMC calibration corruption recovery.
- Tape rotation on disk-full via reachability scan.
- Operator-facing UI for the strategy choice (lands in M22).

## Architecture

### New files

`core/include/ida/ArchivalMode.h`

```cpp
namespace ida
{
/// Per-instance archival strategy for handling plug-in
/// non-determinism. White paper §15.6, §8.3. Default is
/// `VersionPinning` — symbolic fidelity is preserved
/// unconditionally; the performer is warned about plug-in version
/// drift at session reopen and chooses how to react.
enum class ArchivalMode
{
    DeterminismContract,
    WetCapture,
    VersionPinning,
};
} // namespace ida
```

`core/include/ida/VersionPinningRecord.h`
`core/src/VersionPinningRecord.cpp`

```cpp
namespace ida
{
/// The portable, persistable identity of one hosted plug-in instance
/// at the moment it was saved. Round-trips through the session
/// format. Compared on reopen against a freshly computed record; any
/// inequality outside `oversamplingRate` raises a
/// `NotificationBus::PluginVersionDrift` event.
struct VersionPinningRecord
{
    std::string uniqueId;
    std::string version;
    std::string stateBlobSha256;
    std::uint32_t oversamplingRate { 1u };
    std::string declaredInternalStateHash;

    bool matches (const VersionPinningRecord& other) const noexcept;
};

/// Builds the record from the live plug-in instance. For M8 S1, the
/// state blob is the empty byte sequence (no CLAP state integration
/// yet — that arrives in M8 S2); the SHA-256 of zero bytes is well
/// defined and serves as the synthetic-fixture baseline.
VersionPinningRecord makeVersionPinningRecord (
    const PluginDescriptor& descriptor,
    std::span<const std::byte> stateBlob = {});
} // namespace ida
```

The SHA-256 helper used inside `VersionPinningRecord.cpp` is
factored out of `persistence/src/TapeStore.cpp` into a small
`core/src/Sha256.cpp` so `core/` is no longer dependent on
`persistence/` for hashing.

`engine/include/ida/SessionSnapshot.h`
`engine/src/SessionSnapshot.cpp`

```cpp
namespace ida
{
/// Walks `root` and returns a copy-on-write Constituent tree whose
/// EffectChainEntries (with archivalMode == VersionPinning) carry a
/// freshly populated `persistedSnapshot` reflecting the live state
/// of each slot in `host`. Pure: does not mutate either argument.
std::shared_ptr<const Constituent> populateVersionPinningRecords (
    std::shared_ptr<const Constituent> root,
    const OutOfProcessEffectChainHost& host);

/// Walks `loadedRoot` and `liveRoot` (the snapshot at save time vs.
/// the recomputed records from current live state) and pushes a
/// PluginVersionDrift notification per per-entry mismatch.
void verifyVersionPinningOnLoad (
    const Constituent& loadedRoot,
    const OutOfProcessEffectChainHost& host,
    NotificationBus& bus);
} // namespace ida
```

### Modified files

`core/include/ida/PluginDescriptor.h` gains one field:

```cpp
struct PluginDescriptor
{
    PluginFormat format;
    std::string  uniqueId;
    std::string  version;       // new in M8 S1
    std::string  name;
    std::string  manufacturer;
    std::string  filePath;
    // operator== / operator!= unchanged in shape; version added to
    // the comparison.
};
```

JUCE's `PluginDescription::version` already exposes this for
VST3/AU; `host/src/PluginScanner.cpp::fromJuceDescription` populates
it. CLAP descriptors built by hand (the synthetic-CLAP debug button
in `app/MainComponent.cpp::openSyntheticTestPlugin`) populate it
from a hardcoded constant matching the fixture's
`clap_plugin_descriptor->version`.

`core/include/ida/EffectChain.h` gains two fields on
`EffectChainEntry`:

```cpp
struct EffectChainEntry
{
    PluginDescriptor descriptor;
    std::string      displayName;
    bool             bypassed { false };
    ArchivalMode     archivalMode { ArchivalMode::VersionPinning };
    std::optional<VersionPinningRecord> persistedSnapshot;
};
```

`archivalMode` is pure configuration (the strategy chosen for this
slot). `persistedSnapshot` is the freeze of the live plug-in's
identity at the moment of the last `serializeSession` call —
populated by a new engine-side `populateVersionPinningRecords`
helper that walks the Constituent tree and consults the live
`OutOfProcessEffectChainHost` for current descriptor + state per
slot. The walker returns a new Constituent tree because Constituents
are immutable; the snapshot tree is what gets serialized. On
deserialize, `persistedSnapshot` round-trips back populated, and the
engine compares it against a fresh recomputed record at load
completion.

The new helper lives in
`engine/include/ida/SessionSnapshot.h` /
`engine/src/SessionSnapshot.cpp`. It is the bridge between the live
host (which knows the current state) and the persistence layer
(which serializes only configuration).

`engine/src/NotificationBus.cpp` adds a notification kind:

```cpp
struct PluginVersionDrift
{
    ConstituentId        owningConstituent; // the chain's owning node
    std::size_t          slotIndex;         // position within the chain
    VersionPinningRecord expected;          // from persistedSnapshot
    VersionPinningRecord found;             // recomputed at load
};
```

Surfaced via the existing `notificationBus_->push (...)` channel.
The Preparation pane (white paper §6.6 channel-strip surface) is the
natural recipient at runtime; for M8 S1, the notification just lands
on the bus and the existing notification-history pane in
`MainComponent::timerCallback` picks it up — the operator sees a
drift line item without any new UI code.

### Round-trip through the session format

The session format today is JSON; M11 will replace it with IAF.
`persistence/src/SessionFormat.cpp` already serializes
`EffectChainEntry` field-by-field via
`effectChainEntryToVar` / `effectChainEntryFromVar` (lines 503-521).
M8 S1 extends those two functions to round-trip the new
`archivalMode` enum and the optional `persistedSnapshot` record.
The serialization for the record is straightforward — five strings
and one integer — and uses the same `juce::var` / `JSON::toString`
pattern already in the file. The `PluginDescriptor::version` field
added in this session also round-trips through the existing
`pluginDescriptorToVar` / `pluginDescriptorFromVar` pair.

## Behaviour

### Save

`populateVersionPinningRecords(root, host)` runs once before
`serializeSession`:

1. Walk the Constituent tree (read-only).
2. For each `EffectChainEntry` whose
   `archivalMode == VersionPinning`, ask the
   `OutOfProcessEffectChainHost` for the live state blob and current
   descriptor of the matching slot.
3. Build a `VersionPinningRecord` via
   `makeVersionPinningRecord (descriptor, stateBlob)`.
4. Return a copy-on-write tree with `persistedSnapshot` populated on
   those entries. Entries with other `archivalMode` values are
   copied unchanged.

`serializeSession` runs on the returned snapshot tree. Entries with
`archivalMode != VersionPinning` are skipped by the populator —
their records are handled by the (not-yet-implemented)
`DeterminismContract` and `WetCapture` paths in later M8 sessions.

### Load

After `deserializeSession` produces the Constituent tree, the engine
runs a verification pass:

1. Walk the tree (read-only).
2. For each `EffectChainEntry` whose `persistedSnapshot` is
   populated, compute a fresh record from the live slot's current
   descriptor + state blob.
3. Compare via `VersionPinningRecord::matches`. The comparison
   excludes `oversamplingRate` (placeholder until oversampling is
   tracked; comparing the placeholder is meaningless and would
   trigger noise).
4. On any mismatch, push a `PluginVersionDrift` notification with
   the owning `ConstituentId` + slot index + expected/found
   payloads.

The plug-in still loads. White paper line 1500 and V7 line 566 both
specify "warn, do not refuse" — the performer chooses how to react.

### Defaults for new sessions

Every new `EffectChainEntry` defaults `archivalMode` to
`VersionPinning`. This is the V7-prescribed default and matches the
white paper's "symbolic fidelity is never optional" disposition. No
existing session needs migration because the field is constructed
in-place from the default initializer when a session loads from a
JSON document that pre-dates this field.

## Tests

`tests/ArchivalModeTests.cpp` (new).

- **Enum exhaustiveness.** A `switch (mode)` over all three values
  reaches every case; the compiler-warning-as-error in the project's
  baseline catches any future enum extension that forgets to update
  the switch.
- **Record round-trip.** Construct a `VersionPinningRecord` from a
  synthetic descriptor with empty state blob; serialize; deserialize;
  assert equality field-by-field.
- **Hash of empty state.** SHA-256 of zero bytes is
  `e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855`
  (the canonical empty-string SHA-256). Assert this exact value to
  catch any accidental "hash the descriptor itself" regressions.
- **EffectChainEntry round-trip.** Build an entry with
  `archivalMode == VersionPinning` and a manually-populated
  `persistedSnapshot`; pass through
  `effectChainEntryToVar` / `effectChainEntryFromVar`; assert all
  fields survive including the optional snapshot.
- **Verifier — no drift.** Build a Constituent containing one chain
  entry whose `persistedSnapshot` matches the synthetic descriptor;
  run `verifyVersionPinningOnLoad` with a stub `NotificationBus`
  that records pushes; assert zero pushes.
- **Verifier — drift.** Same setup but with the snapshot's `version`
  field mutated to a different string; run the verifier; assert
  exactly one push with the expected/found payload.

`ArchivalMode` tests run entirely on the message thread; no audio
thread surface is exercised. The existing
`MainComponentPluginEditorTests` from M7 S7 stay green — the new
default field on `EffectChainEntry` does not change any code path
those tests exercise. `SessionFormatTests` gains coverage of the new
fields automatically because the round-trip tests above exercise
`effectChainEntryToVar` / `fromVar` directly.

## Risks and open decisions

The major M8-level decisions are already resolved in the V7 plan
(lines 614-618): wet-capture storage budget, CLAP SDK source, trust
model for `isDeterministic`. None apply to this session — they land
in M8 S2, M8 S4, and M8 S5 respectively.

The only S1-local decision worth recording: the `oversamplingRate`
field is included in the record despite the engine not yet tracking
oversampling. Persisting a placeholder is preferable to leaving the
field out and migrating the record format when oversampling lands —
record schemas are easier to extend by populating an unused field
than by adding one in place across deployed sessions. The mismatch
comparison excludes this field for now so the placeholder does not
trigger false drift events.
