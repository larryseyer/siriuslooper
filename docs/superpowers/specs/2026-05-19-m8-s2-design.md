# M8 Session 2 — CLAP scanning + `clap_plugin_state` + live state-blob hashing

## Context

V7 plan line 597 reads: *"CLAP loader in `host_process/main.cpp`;
integration test with a known free CLAP plug-in; commit."* The loader
itself is already shipped (M7 S2c — `runClapMode` does dlopen → entry
init → factory → create_plugin → activate → process). What is actually
missing, and what this session lands, is the surface that lets Sirius
treat CLAP as a first-class plug-in format on par with VST3 and
AudioUnit:

1. **A CLAP scanner** that walks the platform's CLAP search paths and
   produces `PluginDescriptor`s alongside the JUCE-driven VST3 / AU
   scans. JUCE has no first-party CLAP support — the existing
   `PluginScanner` registers only formats JUCE ships natively. The
   loader code that already exists in `host_process/main.cpp`
   (dlopen + `clap_entry` + factory walk) is exactly what a scanner
   needs; extracting it into a shared loader and reusing it from a
   new `ClapScanner` is the bulk of the work.
2. **The `clap_plugin_state` extension** in the child process, so plug-in
   internal state can be saved at session-save time and restored at
   session-load time. M8 S1 left
   `populateVersionPinningRecords`/`verifyVersionPinningOnLoad`
   computing records against the entry's static descriptor with an
   empty byte sequence as the "state blob"; this session fills that in
   with real bytes from a live plug-in.
3. **`OutOfProcessEffectChainHost::descriptorForSlot` and
   `stateBlobForSlot`**, the slot-keyed accessors the M8 S1 followup
   in `todo.md` (~lines 1485-1497) enumerates as the missing wiring.
4. **Drop the `!has_value()` re-pin guard** in
   `engine/src/SessionSnapshot.cpp:48-49` and establish the
   always-refresh + `PluginVersionRepinned` Info-notification
   re-pinning policy so version drift between save-A and save-B is
   visible rather than silent.
5. **Remove the `FabFilter` testing-only scan filter** at
   `host/src/PluginScanner.cpp:39`. It was a M7 S7 stop-gap to keep
   eyes-on testing under a minute against a 1000-plug-in folder;
   replacing it with the real full-scan path while we are in the
   scanner code is cheaper than coming back later. A
   progress-notification per format keeps the "minutes of silence
   during scan" UX problem the filter was working around from
   recurring.

The session deliberately covers both the *persistence* leg (state
hashing + record refresh + notifications) and the *discovery* leg
(scanning + format registration) because they share the same
underlying CLAP-loader code. Splitting them would force a duplicate
extraction of the loader.

This is V7's M8 Session 2 per the M8 block (plan lines 597-598). It
follows M8 S1 (`docs/superpowers/specs/2026-05-19-m8-s1-design.md`,
shipped at HEAD `8d9818f`).

```cpp
namespace sirius
{
/// RAII loader for one .clap bundle. Construct → dlopen + entry init
/// + factory acquisition. Destruct → entry deinit + dlclose, in that
/// strict order. Move-only because the underlying `void*` handle and
/// `clap_plugin_entry_t*` must not be duplicated.
///
/// Threading: construction + destruction are message-thread only.
/// `descriptors()` is const and may be called from any thread that
/// holds a non-mutating reference to the loader.
class ClapBundleLoader
{
public:
    /// Loads the bundle at `bundlePath`. On macOS `bundlePath` is the
    /// `*.clap` bundle directory; the loader resolves the inner
    /// binary at `<bundlePath>/Contents/MacOS/<basename-without-clap>`
    /// (the same logic currently inline at
    /// host_process/main.cpp:350-364). On Linux/Windows `bundlePath`
    /// is the shared library directly.
    ///
    /// `outError` receives a human-readable failure string on any
    /// load step that fails; returned loader is `! valid()`.
    static ClapBundleLoader load (const std::string& bundlePath,
                                  std::string&       outError);

    ~ClapBundleLoader();
    ClapBundleLoader (ClapBundleLoader&&) noexcept;
    ClapBundleLoader& operator= (ClapBundleLoader&&) noexcept;
    ClapBundleLoader (const ClapBundleLoader&)            = delete;
    ClapBundleLoader& operator= (const ClapBundleLoader&) = delete;

    bool                     valid() const noexcept;
    const clap_plugin_entry_t* entry() const noexcept;
    const clap_plugin_factory_t* factory() const noexcept;

    /// Walks the factory and returns one `PluginDescriptor` per
    /// plug-in the bundle exports. Most bundles export exactly one;
    /// shells (e.g. Surge's "Surge XT" bundle exporting both synth
    /// and effects) export several. Non-CLAP fields populated:
    /// format = PluginFormat::Clap, filePath = `bundlePath`,
    /// uniqueId = `<descriptor>->id`, version = `<descriptor>->version`,
    /// name = `<descriptor>->name`,
    /// manufacturer = `<descriptor>->vendor`.
    std::vector<PluginDescriptor> descriptors (const std::string& bundlePath) const;

    /// Instantiates the named plug-in via the factory. Returned
    /// pointer is owned by the bundle's entry — caller must NOT
    /// `delete`; CLAP's `destroy` callback is the disposal path.
    /// Returns nullptr if `pluginId` is not exported by this bundle.
    const clap_plugin_t* createPlugin (const clap_host_t& host,
                                       const char*        pluginId) const;

private:
    ClapBundleLoader();
    void* dlHandle_                       { nullptr };
    const clap_plugin_entry_t*  entry_    { nullptr };
    const clap_plugin_factory_t* factory_ { nullptr };
};
} // namespace sirius
```

namespace sirius
{
/// Non-JUCE CLAP plug-in scanner. Walks the platform's default CLAP
/// search paths (macOS: ~/Library/Audio/Plug-Ins/CLAP +
/// /Library/Audio/Plug-Ins/CLAP; Linux: ~/.clap + /usr/lib/clap +
/// /usr/local/lib/clap; Windows: ~/AppData/Local/Programs/Common/CLAP
/// + C:/Program Files/Common Files/CLAP — per CLAP spec §"Plug-in
/// installation paths") OR a caller-supplied path, and produces a
/// `PluginScanResult` matching the shape `PluginScanner` returns for
/// VST3 / AU.
///
/// Each `*.clap` candidate is loaded through `ClapBundleLoader`;
/// failures populate `result.failedFiles` rather than throwing.
/// Message-thread only — `dlopen` allocates and CLAP `entry->init`
/// may take seconds for heavyweight plug-ins.
class ClapScanner
{
public:
    /// Returns the platform's CLAP search paths in priority order.
    static std::vector<juce::File> defaultSearchPaths();

    /// Scans `path` recursively for `*.clap` bundles, returning a
    /// `PluginScanResult` populated with descriptors + failed files.
    PluginScanResult scan (const juce::File& path);

    /// Convenience — scans every path returned by `defaultSearchPaths`.
    PluginScanResult scanDefaultLocations();
};
} // namespace sirius

`host/include/ida/PluginStateRegion.h`
`host/src/PluginStateRegion.cpp`

```cpp
namespace sirius
{
/// Fixed-layout shared-memory struct backing the
/// `state.<instanceId>` POSIX shm region. Sized for the largest
/// realistic CLAP state blob we expect to round-trip in one IPC
/// hop (kMaxStateBytes = 64 KiB — covers everything FabFilter-class
/// EQs serialize and ~95% of synths; plug-ins emitting larger
/// states will be addressed when we hit one).
///
/// Request/response sequencing mirrors `PluginGuiState`: the
/// engine increments `requestSeq` after writing the request fields
/// and the kind; the child compares against `lastServicedSeq` on
/// every idle tick, services the request, fills the response
/// fields, then publishes `responseSeq = current` with release
/// ordering. The engine spins on `responseSeq` with a deadline
/// (default 250 ms in `requestStateSave` / `requestStateLoad`).
struct PluginStateState
{
    enum Kind : std::uint32_t { None = 0, Save = 1, Load = 2 };

    static constexpr std::size_t kMaxStateBytes = 64u * 1024u;

    std::atomic<std::uint64_t> requestSeq    { 0 };
    std::atomic<std::uint32_t> requestKind   { None };
    std::atomic<std::uint32_t> requestBytes  { 0 };       // for Load
    char                       requestPayload [kMaxStateBytes] {}; // for Load

    std::atomic<std::uint64_t> responseSeq        { 0 };
    std::atomic<std::uint32_t> responseStatus     { 0 };  // 0=ok, 1=err
    std::atomic<std::uint32_t> responseBytes      { 0 };  // for Save
    char                       responsePayload [kMaxStateBytes] {}; // for Save

    /// Region size — used by the engine when creating the region and
    /// by the child when attaching.
    static constexpr std::size_t kRegionBytes = sizeof (PluginStateState);

    /// Constructs the in-place view over a previously-mmaped region.
    /// Same shape as `PluginGuiState::view`.
    static PluginStateState* view (void* mapped) noexcept;
};

std::string makeStateRegionName (const std::string& instanceId);
} // namespace sirius
```

## Behaviour

### Scan path

`PluginScanner::scanDefaultLocations` now runs three passes: JUCE
VST3, JUCE AU (macOS only), and the new `ClapScanner`. Results
union into one `PluginScanResult`. Operator sees CLAP plug-ins in
the plug-ins pane alongside VST3 / AU with no UI changes — the
existing list is format-agnostic.

`ClapScanner::scan` walks the path tree (recursive), filtering for
entries matching `*.clap` (a directory bundle on macOS, a shared
library elsewhere). For each match it constructs a
`ClapBundleLoader`; on failure the bundle path is added to
`result.failedFiles` and the loader is discarded. On success the
loader's `descriptors(bundlePath)` is appended to
`result.descriptors`. The loader's RAII destructor `deinit`s the
entry + `dlclose`s the handle between bundles, so scanning a
1000-bundle folder doesn't leak handles.

A progress notification (Info / PluginEvent) is posted per format
at start and end. The scan stays synchronous — V7 leaves async
scanning to a later milestone, and the operator's current scan
volume (no auto-scan, only operator-triggered) doesn't warrant a
thread pool.

### Save path

`MainComponent::chooseFileAndSave` (or whichever call site M8 S1
wired) now:

1. Computes the populated snapshot tree via
   `populateVersionPinningRecords (rootNode, effectChainHost_, lookup)`.
2. The populator walks the tree. For every `EffectChainEntry` with
   `archivalMode == VersionPinning`:
   - Resolves `(busId, slotIndex)` via the lookup callback.
   - Asks the host for the live descriptor via
     `descriptorForSlot (busId, slotIndex)`.
   - Asks the host for the live state blob via
     `stateBlobForSlot (busId, slotIndex)` (250 ms timeout;
     timeout → empty blob fallback + Warning notification).
   - Builds a fresh `VersionPinningRecord` via
     `makeVersionPinningRecord (descriptor, stateBlob)`.
   - If the entry already had a `persistedSnapshot` and the fresh
     record's `matches()` returns false against the prior, posts
     `Info / PluginEvent / PluginVersionRepinned`.
   - Sets the entry's `persistedSnapshot` to the fresh record
     (always-refresh policy).
3. Serializes the populated tree as before.

No save ever fails because of version drift. The notification gives
the operator a record without blocking the gesture.

### Load path

`MainComponent::chooseFileAndLoad` runs the existing M8 S1
verification pass after `deserializeSession`:

1. `verifyVersionPinningOnLoad (loadedRoot, effectChainHost_,
   sink, lookup)` walks the tree.
2. For each `EffectChainEntry` with a populated `persistedSnapshot`,
   it computes a fresh record from the live slot's current
   descriptor + state blob.
3. On mismatch via `matches()`, posts
   `Warning / PluginEvent / PluginVersionDrift` with the
   new hash-bearing message format.

Operator sees a drift line in the notification-history pane; the
plug-in still loads (per white paper line 1500 and V7 line 566 —
"warn, do not refuse").

### Re-pin notification format

```
"plug-in version repinned: %s old=%s new=%s"
```

Same byte-budget arithmetic as the drift message but lighter
(only versions, no hashes, because the repin event itself signals
state change; the operator's interest is "did the version
change?").

### Drift notification format

```
"version drift: %s ev=%s eh=%s fv=%s fh=%s"
```

Byte-budget arithmetic (target ≤127 chars + 1 null):

- Fixed prefix `version drift: ` = 15 chars.
- `uniqueId` typical length: 40-60 chars for Java-style reverse-DNS
  IDs (e.g. `com.fabfilter.proq3`), up to ~80 for vendor-prefixed
  ones.
- ` ev=` + version (typical 3-8 chars) = ~10 chars × 2 = 20.
- ` eh=` + 8 hex chars × 2 = 24 chars.
- Separators total: 4.

Worst-case-typical (80-char uniqueId): 15 + 80 + 20 + 24 + 4 = 143.
Over budget for 80-char uniqueIds; the `snprintf` will truncate at
127. Truncation happens from the right, so under the new format
the loss order is `fh` → `fv` → `eh` → `ev` → `uniqueId`. The
*hash* is the most-actionable field — different versions reporting
the same hash means a vendor mislabel, identical versions with
different hashes means undeclared internal-state change. Reorder
the format to put the hash pair first:

```
"version drift eh=%s fh=%s ev=%s fv=%s id=%s"
```

Fixed prefix `version drift eh=` = 17 chars, then 8 hex, ` fh=` + 8
hex = 13 chars, ` ev=` + version = 11 chars, ` fv=` + version = 11
chars, ` id=` + uniqueId = up to 84 chars. Worst-case typical:
17 + 8 + 13 + 11 + 11 + 84 = 144 — still over by 17 chars on a
maximal uniqueId, but the truncation now eats the uniqueId tail
(usually the unique vendor suffix is preserved because reverse-DNS
puts the vendor prefix at the start, e.g.
`com.fabfilter.` → truncation eats `proq3...`). Acceptable: the
hashes and versions, both fully diagnostic, always survive.

### Re-pinning policy rationale

Three options were considered:

1. **Refuse to save when version drift is detected.** Rejected as a
   footgun. Save must never fail or block from the operator's
   perspective — losing a take to a dialog box is an unforgivable
   pro-audio sin. White paper line 1500 already commits to "warn,
   do not refuse" for load; the same posture applies to save by
   symmetry.
2. **Silently refresh on every save.** Rejected because a
   v1.0.0→v1.0.1 upgrade between save-A and save-B disappears
   into the new record with no operator awareness. Sessions become
   subtly inconsistent across machines.
3. **Always refresh + post Info notification when the record
   changes (chosen).** Pro-audio convention: the operator sees what
   happened, the gesture completes, the next save has the new pin.
   Notification is Info-level (not Warning) because a deliberate
   upgrade is not a problem to be alarmed about; the operator
   filters by level if they want a clean history.

### CLAP integration test automation

The synthetic test plug-in is a stereo identity passthrough — it
exercises the IPC and audio paths but not the third-party-shaped
surface (parameters, internal state, multi-instance, multiple
descriptors per bundle). To cover that surface in CI without an
external dependency, the new `StatefulSynthFixture` is a second
synthetic CLAP shaped like a real third-party plug-in: distinct
ID, two parameters, real internal state, real `clap_plugin_state`
serialization. Tests instantiate it, drive the parameter changes,
call `requestStateSave`, destroy and re-create the instance, call
`requestStateLoad`, and assert the parameters round-tripped.

For real-world FabFilter-class validation, the new
`IDA_THIRDPARTY_CLAP_PATH` CMake variable lets the operator
point at any installed CLAP bundle. When set, the test suite
registers a `ThirdPartyClapIntegrationTest` ctest case that runs
the same instantiate / parameter / save / load cycle against the
real bundle. When unset (CI, default dev), the test is silently
absent. Operator binds the variable once via CMake cache; every
subsequent build runs the validation.

## Risks and open decisions

**`PluginStateState::kMaxStateBytes = 64 KiB`** — adequate for
EQs, compressors, and most synths. Wavetable synths or sampler
plug-ins that serialize embedded samples will exceed this. The
out-of-bounds case currently returns an error from `requestStateSave`
(the child detects `outBytes.size() > kMaxStateBytes` and writes
`responseStatus = 1`). When we encounter a real plug-in this
matters for, we add a streaming/chunked protocol — single-shot for
small states, multi-message for large. Not in M8 S2 scope.

**Drift-message truncation under maximal uniqueIds.** The reordered
format preserves the two most-diagnostic fields (state hashes) at
all uniqueId lengths. The uniqueId tail truncates at ~84 chars;
operators with 100+ char vendor-prefixed IDs lose the suffix. The
prior 128-byte notification buffer is a `NotificationBus` invariant
shared by all notification kinds; widening it for one kind has
ripple effects out of scope here.

**`PluginScanner::setNotificationSink` introduces a setter on a
class M7 leaves sink-less.** The two existing call sites
(`scanDefaultLocations`, `scan`) currently take no sink. Adding the
setter is purely additive; existing tests pass nullptr by virtue
of the field defaulting to nullptr. Documented as a deliberate
choice in the header comment.

**`IDA_THIRDPARTY_CLAP_PATH` test depends on operator-supplied
state.** When the operator updates their FabFilter version, the
test's cached state bytes (if any are committed) go stale. The
test does not commit cached state — it round-trips bytes through
the live plug-in in-process — so this is a non-issue today; flagged
for the future case where we add a save-known-good-bytes step.

**CLAP search paths beyond the defaults.** Some plug-in vendors
install to `/Applications/<Vendor>/CLAP/` or similar. The CLAP
spec only mandates the standard paths; the operator's custom paths
land via the existing `PluginScanner::scan (const juce::FileSearchPath&)`
overload, which the CLAP scanner inherits via the format-loop. UI
for managing custom paths is M22 surface; out of scope here.
