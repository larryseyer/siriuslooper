# M8 S4 — WetCapture writer

## Problem

`ArchivalMode::WetCapture` has existed as an enum value since M8 S1, but nothing
produces or consumes wet tapes. `SessionSnapshot::populateVersionPinningRecords`
(`engine/src/SessionSnapshot.cpp`) deliberately skips WetCapture entries with a
comment deferring them to "M8 S4-5". V7 alignment (line 565) defines the
acceptance: WetCapture mode adds a `<channelId>.wet.tape` alongside the dry tape,
stored in the SAF `tapes/` subdir.

WetCapture is the archival strategy that survives plug-in loss: instead of
re-running a plug-in deterministically (DeterminismContract) or pinning its
version and warning on drift (VersionPinning), it records the actual processed
(wet) audio output, so the session reproduces regardless of whether the plug-in
still loads. The wet tape is the post-effects counterpart of the dry tape.

## Scope

The dry tape (`TapeWriter`) captures *input* recordings keyed by `InputMixer`
`ChannelId`. Wet (post-effects) audio only exists in `Bus::processedBuffer_` on
the *output* side, where effect chains run per-`EffectChainEntry`. Per-Constituent
channel strips (whitepaper §6.6) and `RenderPipeline`-in-the-audio-callback are
not wired (M9+ / M8 S3 honest-scope notes), so a fully-wired "capture each
loop's post-effects output, keyed to its Constituent" is not buildable without
machinery V7 sequences later.

This session ships the **mechanism plus a tested seam**, matching the M8 S1/S3
precedent (M8 S3 shipped the 3-arg `RenderPipeline` ctor as a real, tested seam
with audio-callback wiring deferred).

In:

- `WetCaptureWriter` — a complete, tested capture pipeline parallel to the M3
  `TapeWriter`: lock-free audio-thread enqueue → worker-thread drain to a
  `<channelId>.wet.tape.partial` working file → finalize into `TapeStore`
  (content-addressed `<sha256>.tape`).
- `IWetCaptureSink` — the audio-thread interface `Bus` depends on (parallel to
  how `Bus` depends on `IEffectChainHost`, not the concrete host).
- A default-off, RT-safe tap seam on `Bus`: when a wet-capture sink is set,
  `Bus::process` enqueues `processedBuffer_` (the post-effects signal) after the
  effect chain. Zero overhead when unset.
- Tests covering the writer in isolation and the Bus seam end-to-end.

Out (deferred, recorded in `todo.md`):

- Binding the Bus seam to a production capture-arm flow. The seam is set-once
  plumbing this session; nothing in the operator flow wires it yet.
- Per-Constituent effect-chain routing (M9+) and the structure-layer
  `TapeId → wet-tape hash` mapping (M11 SAF) — exactly as the dry tape's
  structure mapping is already deferred to M11.
- `SessionSnapshot` WetCapture handling (still correctly skipped; wet tapes are
  captured by this writer, not by snapshotting).
- Extracting a shared partial-writer core from `TapeWriter` + `WetCaptureWriter`
  — would touch proven M3 RT code; deferred to a dedicated refactor.

## Key constraints

- **RT-safety** (`docs/RT_SAFETY_CONTRACT.md`): `tryEnqueueWet` and
  `Bus::process` are `noexcept`, allocation-free, lock-free, I/O-free.
  Queue-full / oversized-buffer returns `false`; the caller treats it as a miss
  (no overload escalation is added to `Bus` this session — the seam is not yet a
  production capture path).
- **Bus collaborator contract**: the wet sink is a non-owning, set-once pointer
  installed on the message thread before the audio thread starts, mirroring
  `Bus::setEffectChainHost`. `Bus::process` is `const noexcept`; it reads the
  pointer value only.
- **No changes to `TapeWriter`.** Proven M3 RT code stays untouched. The hard
  concurrency primitive (`LockFreeSpscQueue<T>`) is already shared infra; only
  the thin worker-drain loop is parallel.
- **Message size cap** mirrors `kMaxTapeWriteMessageBytes` (32 KB → 4096 stereo
  float32 samples). Buffers exceeding the cap return `false`, matching the dry
  path's per-message ceiling.

## Design

### `IWetCaptureSink` — `engine/include/sirius/IWetCaptureSink.h`

Pure interface, JUCE-free (like `IEffectChainHost`):

```cpp
class IWetCaptureSink
{
public:
    virtual ~IWetCaptureSink() = default;

    /// Audio-thread entry. Planar input: `channels[c]` points at `numSamples`
    /// float32 for channel c. Returns true on enqueue, false on queue-full or
    /// oversized buffer. Wait-free, allocation-free, I/O-free.
    [[nodiscard]] virtual bool tryEnqueueWet (ChannelId id,
                                              Rational lmcTime,
                                              const float* const* channels,
                                              int numChannels,
                                              int numSamples) noexcept = 0;
};
```

Planar `const float* const*` matches `Bus::process` / `IEffectChainHost`; the
sink interleaves into its own message, so `Bus` never names the wire layout.

### `WetCaptureWriter` — `engine/include/sirius/WetCaptureWriter.{h,cpp}`

Implements `IWetCaptureSink`. Structure mirrors `TapeWriter`:

- ctor `(std::filesystem::path partialDir, std::chrono::milliseconds flushInterval, std::size_t queueCapacity)` — constructs a `LockFreeSpscQueue<WetWriteMessage>` and starts the worker thread.
- `WetWriteMessage` — POD, same shape as `TapeWriteMessage`: `ChannelId id`,
  `Rational lmcTime`, `std::size_t payloadByteCount`,
  `std::array<std::byte, kMaxTapeWriteMessageBytes> samples`. The payload is
  interleaved float32: `[ch0 s0, ch1 s0, ch0 s1, ch1 s1, ...]`.
- `tryEnqueueWet(...)` — interleaves the planar input into a stack
  `WetWriteMessage`; if the interleaved byte count exceeds the cap, returns
  `false` without enqueuing; otherwise pushes via the queue and returns its
  result. Wait-free.
- Worker thread drains to `partialDir / <channelId>.wet.tape.partial` via the
  same open-file / flush-interval model as `TapeWriter`.
- `flushChannel(ChannelId)` — blocks until the worker has flushed all pending
  messages for the channel and closed the file; returns the partial path (which
  may not exist if nothing was enqueued — caller checks).
- `finalizeToStore(ChannelId, persistence::TapeStore&)` — message-thread helper:
  `flushChannel`, read the partial into a `juce::MemoryBlock`, `TapeStore::store`
  it, return the content hash (`juce::String`). Returns an empty string if the
  partial does not exist (nothing captured). Deletes the partial after store.
- `setNotificationBus(NotificationBus*)` — on worker-thread I/O failure, posts
  `Error / Category::DiskPressure` and bumps a per-channel error counter, exactly
  as `TapeWriter` does.
- `errorCountForChannel(ChannelId)` — diagnostics accessor, parallel to
  `TapeWriter`.

### `Bus` seam — `engine/include/sirius/Bus.h` + `engine/src/Bus.cpp`

- New member: `IWetCaptureSink* wetSink_ { nullptr }` and `ChannelId
  wetCaptureId_ { 0 }`.
- New message-thread setter: `setWetCaptureSink(IWetCaptureSink* sink, ChannelId
  captureId) noexcept` — non-owning, set-once before audio starts, mirroring
  `setEffectChainHost`. `nullptr` disables (default).
- In `Bus::process`, on the effect-chain path, after the chain has filled
  `processedBuffer_` and **before** it is summed into `output`, if `wetSink_` is
  set: build planar channel pointers into `processedBuffer_`
  (`processedBuffer_.data() + c * kMaxBusMixSamples`, up to the active channel
  count) and call `wetSink_->tryEnqueueWet(wetCaptureId_, Rational{0}, chans,
  numChannels, numSamples)`. `lmcTime = 0` parallels the dry path (no per-channel
  LMC context yet). The return value is ignored (a miss simply drops that
  buffer, same backpressure semantics as the dry tape). A single null-check
  branch; the inline (no-host) path and the unset-sink case are unaffected.

### Data flow

`Bus::process` runs the effect chain → wet audio in `processedBuffer_` → (sink
set?) `tryEnqueueWet` → worker writes `<channelId>.wet.tape.partial` →
`finalizeToStore` → content-addressed `<sha256>.tape` in `tapes/`. Identical
lifecycle to the dry tape; the structure-layer `TapeId → hash` mapping is M11.

## Testing — `tests/WetCaptureWriterTests.cpp`, tag `[wet-capture]`

1. **Enqueue → flush → content.** Enqueue a known planar buffer; after worker
   flush, the `<channelId>.wet.tape.partial` bytes equal the expected
   interleaved float32.
2. **`flushChannel` completeness.** After `flushChannel`, the returned partial is
   closed and contains every enqueued message in order.
3. **Finalize round-trip.** `finalizeToStore` returns a non-empty hash; the
   stored tape reads back via `TapeStore::read` byte-for-byte; the partial is
   gone.
4. **Oversized buffer rejected.** A buffer whose interleaved size exceeds the cap
   makes `tryEnqueueWet` return `false` and enqueues nothing.
5. **I/O failure surfaces.** With an unwritable `partialDir` and a
   `NotificationBus` attached, a flush failure posts `Error / DiskPressure` and
   bumps `errorCountForChannel`, without crashing.
6. **Bus seam end-to-end.** A real `Bus` with a bound `IEffectChainHost`
   (the test effect-chain host the existing Bus tests use) and at least one
   non-bypassed slot, plus a real
   `WetCaptureWriter` sink: populate `mixBuffer_`, `process`, then
   `finalizeToStore` and assert the captured tape reflects the post-effects
   signal (distinct from the dry input, proving the tap point is post-chain).
7. **Unset sink is inert.** A `Bus` with no wet sink behaves bit-for-bit as
   before (regression guard on the hot path).
